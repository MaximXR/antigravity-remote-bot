# Системные правила разработки Remoat

## 1. База Данных и Контекст

## 2. Производительность CDP (КРИТИЧЕСКИ)
- **ЗАПРЕЩЕНО** использовать неспецифичные селекторы общего назначения (`div`, `span`, `*`) для фоновых периодических опросов по всей странице. Это вызывает сильные лаги интерфейса.
- **ОБЯЗАТЕЛЬНО** сужать выборку на уровне браузера через специфичные теги/классы/роли (например: `button`, `[role="button"]`, `.cursor-pointer`).
- **ОБЯЗАТЕЛЬНО** ставить фоновые детекторы на паузу на время активной генерации ответа ИИ (когда работает `ResponseMonitor`).
- **ОБЯЗАТЕЛЬНО** оборачивать фоновые JS-скрипты опроса в `try {} catch (e) {}` для стабильности цикла опроса.

## 3. Нормализация кнопок и нечеткий поиск
- Метод `buildClickScript` приводит все целевые тексты кнопок к **нижнему регистру**.
- **ОБЯЗАТЕЛЬНО** использовать нижний регистр (`allow`, `deny`, `allow this conversation`) во всех тест-моках и CDP-проверках кликов.
- При поиске сессии по имени в IDE сравнивать первые 20–25 символов без учета регистра по взаимному вхождению: `wanted.includes(itemTitle)` или `itemTitle.includes(wanted)`.

## 4. Разделение слоев (Bot Layer / Core Layer)
- Логика мессенджера (работа с `grammy`, форматирование сообщений под лимиты Telegram, клавиатуры) изолирована в `src/bot/index.ts`.
- Логика IDE и CDP (инжекция, мониторинг, скриншоты) инкапсулирована в `src/services/idePromptRunner.ts` и `src/services/quickPickResolver.ts`.
- Взаимодействие между слоями выполняется строго через коллбеки `IdePromptCallbacks`.
- **ОБЯЗАТЕЛЬНО** сохранять настройки трансляции диалогов (`mirrorMode`, `onlyActiveWorkspaceMessages`) в слое бота при фильтрации сообщений.

## 6. Принцип DRY (Don't Repeat Yourself) (КРИТИЧЕСКИ)
- **КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО** дублировать код обработчиков команд Telegram-бота и бизнес-логику.
- Любое дублирование кода неизбежно ведет к регрессионным багам: *«в одном месте чинишь — в другом ломается»*.
- **ОБЯЗАТЕЛЬНО** выносить все обработчики команд бота исключительно в файл [commands.ts](<workspace-root>/src/bot/commands.ts) (или подключать из `src/commands/`).
- В файле [index.ts](<workspace-root>/src/bot/index.ts) разрешено только импортировать и регистрировать команды через соответствующие функции-инициализаторы.
- *Контрастный пример:*
  - ❌ **Как НЕ надо:** Описывать команду `/status` параллельно в `index.ts` и `commands.ts`.
  - ✅ **Как надо:** Объявить обработчик `/status` в `commands.ts`, а в `index.ts` вызвать `registerCommands(bot, deps)`.
